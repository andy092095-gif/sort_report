## 一、 前言
在計算機科學領域中，資料排序（Sorting）是一項不可或缺的核心技術，廣泛應用於資料處理、搜尋最佳化及各類進階演算法的底層架構中。本報告聚焦於探討數種經典的排序演算法，包含：氣泡排序（Bubble Sort）、選擇排序（Selection Sort）、插入排序（Insertion Sort）、快速排序（Quick Sort）以及合併排序（Merge Sort）。我們將從時間複雜度（Time Complexity）與空間複雜度的角度出發，結合理論推導與實際程式測試，深入比較各演算法在不同資料規模與情境下的運算效率與優劣勢。
## 二、 排序法原理介紹

### 1. 氣泡排序 (Bubble Sort) 🫧
* **原理**：兩兩比較相鄰的資料，若順序錯誤就互相對調。這就像水裡的泡泡一樣，透過一輪又一輪的推擠，讓最大（或最小）的數值慢慢滾到陣列的最後方。
* **特性**：概念最簡單、寫法最直覺，但因為需要進行大量的換位動作，執行時間會拖得很長。通常只作為程式初學者的教學示範，很少在實務上使用。

### 2. 插入排序 (Insertion Sort) 📥
* **原理**：想像你在打撲克牌時整理手牌，把還沒排好的數字一張一張抽出來，往前尋找合適的大小位置，然後安插進去。
* **特性**：如果資料「已經大致排好序」，它的速度會飛快！另外它是一種穩定的排序方式，就算有兩個一樣的數字，排完後也不會打亂它們原本的先後順序。

### 3. 選擇排序 (Selection Sort) 🎯
* **原理**：每一回合都把整個未排序的區間掃描一遍，一眼挑出當中的最小值（或最大值），然後直接把它抓到前面，跟未排序區間的第一個元素互換位置。
* **特性**：最大的優點是「交換位置」的動作非常少。但缺點是，無論原本資料是整齊還是雜亂，它都很死腦筋地需要花費一樣多的時間去尋找極值，整體效能依然不佳。

### 4. 合併排序 (Merge Sort) 🔀
* **原理**：核心精神是「把大問題切成小問題」。先將整組資料不斷對半劈開，直到剩下單一個體；接著再將這些個體兩兩比對大小，層層往上合併，最後就會拼湊回排序好的完整陣列。
* **特性**：無論遇到什麼樣的資料分布，都能保證非常優秀且穩定的運算速度。缺點是合併的過程中，必須額外借用與原始資料同等大小的「記憶體空間」來當作暫存區。

### 5. 快速排序 (Quick Sort) 🔢
* **原理**：隨機挑選一個數字當作「基準點」，接著像分邊站一樣，把比它小的數字全丟到左邊，比它大的全丟到右邊。然後對左右兩半邊重複同樣的分類動作，直到全部排完。
* **特性**：在多數真實應用場景中速度稱霸，且極度節省空間，是業界最愛用的演算法。但若運氣極差選錯基準點（例如資料已經是正序，你又選到最大值），速度會退化得很嚴重。

## 三、 複雜度理論分析
| 排序演算法 | 時間複雜度 (平均) | 空間複雜度 | 穩定性 | 測試時間 (ms) |
| :--- | :--- | :--- | :--- | :--- |
| **氣泡排序** | $O(n^2)$ | $O(1)$ | 穩定 | - |
| **選擇排序** | $O(n^2)$ | $O(1)$ | 不穩定 | - |
| **插入排序** | $O(n^2)$ | $O(1)$ | 穩定 | - |
| **合併排序** | $O(n \log n)$ | $O(n)$ | 穩定 | - |
| **快速排序** | $O(n \log n)$ | $O(\log n)$ | 不穩定 | - |

## 四、 實驗設計與數據模擬

### 1. 實驗環境
* **實作語言**：JavaScript (基於 `performance.now()` 高精度計時)。
* **數據類型**：隨機排列 (Random)、近似排序 (Nearly-sorted)、倒序排列 (Reverse)。
* **計時方式**：純算法計時，排除視覺化與 DOM 操作干擾。

### 2. 實驗變因
* **操作變因 ($n$)**：數據規模，範圍從 100 到 10,000。
* **應變變因**：執行時間（毫秒 $ms$）。
* **重複次數**：同數據集多次測試取平均值以消除系統干擾。

## 五、 實驗結果呈現與比較

### 1. 執行時間趨勢表 (模擬數據)
| 演算法 | n=500 | n=1000 | n=2000 | n=5000 | n=10000 | 平均時間 | 增長倍率 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| **氣泡排序** | 2.200 ms | 5.100 ms | 2.667 ms | 16.500 ms | 92.600 ms | **23.813 ms** | **42.09x** |
| **選擇排序** | 0.467 ms | 0.733 ms | 2.167 ms | 8.033 ms | 33.567 ms | **8.993 ms** | **71.93x** |
| **插入排序** | 0.300 ms | 0.233 ms | 0.733 ms | 4.933 ms | 21.367 ms | **5.513 ms** | **71.22x** |
| **合併排序** | 0.400 ms | 0.300 ms | 0.500 ms | 1.633 ms | 4.133 ms | **1.393 ms** | **10.33x** |
| **快速排序** | 0.567 ms | 0.767 ms | 1.700 ms | 2.767 ms | 3.167 ms | **1.793 ms** | **5.59x** |

### 2. 效能差異分析
**1. 二次方程式量級 $O(n^2)$ 家族包含：氣泡排序、選擇排序、插入排序特徵**：當資料量 $n$ 變大時，執行時間會呈現拋物線式的攀升。你可以看到在 $n=5000$ 到 $n=10000$ 這個區間，這三者的耗時都有顯著的「爆發性」成長。氣泡排序 (Bubble Sort)：效能最差。雖然 $n=2000$ 時的數據有點異常（可能是測量誤差或系統快取影響），但到了 $n=10000$ 時耗時高達 92.6 ms，是所有演算法中最慢的。選擇排序 (Selection Sort) 與 插入排序 (Insertion Sort)：這兩者在處理大數據時的成長倍率（約 71 倍）也很驚人。不過在絕對時間上，插入排序通常比選擇和氣泡快一點，尤其是當遇到「部分已排序」的資料時，插入排序的表現會出乎意料的好。

**2. 對數線數量級 $O(n \log n)$ 家族包含：合併排序、快速排序特徵**：這類演算法的成長曲線平緩很多，非常適合處理大型資料集。合併排序 (Merge Sort)：表現非常穩定。不管資料的初始亂度如何，它都能保持 $O(n \log n)$ 的水準，在 $n=10000$ 時只需要 4.133 ms。快速排序 (Quick Sort)：這份圖表中的效能冠軍。在 $n=10000$ 時只需 3.167 ms，且整體增長倍率最低（5.59x）。在我們寫 C/C++ 時，標準函式庫 <algorithm> 中的 std::sort 底層主要就是使用優化過的快速排序（IntroSort），這也是為什麼在實務開發中，我們幾乎都是直接呼叫內建函式，因為它的效率極高。



## 六、 結論與建議

本次測試完美驗證了演算法時間複雜度的理論差異。氣泡、選擇與插入等 O(n^2) 基礎演算法在資料量增長時耗時劇烈攀升，不適合處理大量數據。相反地，合併與快速排序等 O(n log n) 演算法展現出極佳的擴展性，在萬筆資料下仍能維持 4 ms 以內的極速表現。實務建議上，開發時面對大規模數據應優先採用增長倍率最低的快速排序或穩定的合併排序。至於插入排序則因為常數開銷小，適合做為處理微型資料時的輕量化選擇。
